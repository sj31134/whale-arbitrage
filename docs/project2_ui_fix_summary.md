## 🔧 수정된 문제

### 1. 날짜 범위 에러
**문제**: `to_date`의 기본값이 `max_value`보다 큰 경우 에러 발생
```
StreamlitAPIException: The default value of [datetime.date(2025, 11, 23)] 
must lie between the min_value of 2023-12-30 and the max_value of 2025-11-22
```

**해결**:
- `to_date` 기본값을 `max_date_obj`로 설정
- 날짜 범위 검증 로직 추가

### 2. UI 구성 문제

#### 차익거래 계산 페이지
**문제**: 
- 입력 파라미터가 사이드바에 제대로 표시되지 않음
- `with st.sidebar:` 블록 사용으로 인한 들여쓰기 문제

**해결**:
- 모든 입력 요소를 `st.sidebar.*`로 직접 호출
- 섹션별로 `st.sidebar.subheader()` 사용하여 구조화

#### 최적 전략 추천 페이지
**문제**: 
- 입력 파라미터가 사이드바에 표시되지 않음

**해결**:
- 모든 입력 요소를 `st.sidebar.*`로 직접 호출
- 섹션별 구조화
